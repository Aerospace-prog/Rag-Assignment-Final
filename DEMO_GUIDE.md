# Video Demonstration Guide

## Overview

This guide provides step-by-step instructions for recording the assignment video demonstration of the Stock Market & Investment Analysis RAG System. Follow these instructions carefully to ensure your video meets all submission requirements.

**Total Duration**: 8-10 minutes  
**Submission Format**: Name_RollNumber.mp4  
**Deadline**: March 5, 2026

## Prerequisites

Before starting your recording, ensure:

- ✅ Production deployment is live and verified
- ✅ Investment Book PDF is ready for upload
- ✅ Firebase Console is accessible (logged in)
- ✅ Screen recording software is set up and tested
- ✅ Webcam is positioned and visible
- ✅ Audio is clear (test with a short recording)
- ✅ Browser tabs are organized (application tab, Firebase Console tab)
- ✅ All 5 mandatory queries have been tested and work correctly

## Demonstration Flow

### 1. Introduction (30 seconds)

**What to show:**
- Your webcam (must be visible throughout the entire video)
- Introduce yourself: name and roll number
- Brief project description: "Stock Market & Investment Analysis RAG System"
- Show the production application URL in the browser address bar

**Script example:**
> "Hello, I'm [Your Name], roll number [Your Roll Number]. This is my Stock Market & Investment Analysis RAG System, a Retrieval-Augmented Generation application that uses vector embeddings and semantic search to answer questions about investment and stock market concepts."

### 2. PDF Upload Demonstration (1 minute)

**What to show:**
1. Navigate to the main application interface
2. Click the upload button (show cursor clearly)
3. Select the Investment Book PDF from your file system
4. Show the upload progress indicator
5. Verify the document appears in the sidebar/document list
6. Mention that the system is now processing the document

**Script example:**
> "I'll now upload the Investment Book PDF. The system will extract the text, chunk it into smaller segments, and generate vector embeddings for semantic search."

**Important:**
- Speak while the upload is in progress
- Don't skip ahead - show the actual upload process
- Point out when the document appears in the UI

### 3. Firebase Console - Document Chunks (1 minute)

**What to show:**
1. Open Firebase Console in a new browser tab (keep application tab open)
2. Navigate to Firestore Database
3. Show the "document_chunks" collection
4. Click on one chunk document to expand it
5. Scroll through and point out all required fields:
   - `text` (the actual chunk content)
   - `chunk_index` (position in document)
   - `page_number` (source page)
   - `document_name` (Investment Book)
   - `filename` (PDF filename)
   - `user_email` (your email)
   - `status` (should be "processed")
   - `char_count` (character count)
   - `created_at` (timestamp)
   - `doc_type` (document type)

**Script example:**
> "In the Firebase Console, you can see the document_chunks collection. Each chunk contains the text content, metadata like chunk index and page number, and the processing status. This chunk has been processed and is ready for semantic search."

### 4. Firebase Console - Embeddings (1 minute)

**What to show:**
1. In the same chunk document, scroll to the `embedding` field
2. Click to expand the embedding array
3. Show the numerical values (1536-dimensional vector)
4. Scroll through some of the values to demonstrate the array length
5. Navigate to at least one more chunk and show its embedding field
6. Explain that these are vector representations generated by Gemini

**Script example:**
> "Here's the embedding field - a 1536-dimensional vector generated by Google's Gemini model. These numerical values represent the semantic meaning of the text chunk. I'll show another chunk to demonstrate that multiple chunks have been successfully embedded."

**Important:**
- Show at least 2 different chunks with visible embeddings
- Expand the array to show actual numerical values
- Don't just show the field name - show the content

## Recording Tips

### Technical Setup
- **Screen Resolution**: Use 1920x1080 or 1280x720 for best quality
- **Webcam Position**: Bottom-right or top-right corner, small but visible
- **Audio**: Use a good microphone or headset, minimize background noise
- **Browser Zoom**: Set to 100% for clarity
- **Close Unnecessary Tabs**: Keep only application and Firebase Console tabs open

### During Recording
- **Speak Clearly**: Moderate pace, not too fast
- **Show Your Actions**: Move cursor deliberately, don't rush clicks
- **Scroll Slowly**: Especially for answers - viewers need to read
- **Pause Briefly**: After each query response, pause 1-2 seconds before scrolling
- **Stay Calm**: If you make a mistake, pause and restart that section

### Common Mistakes to Avoid
- ❌ Webcam not visible throughout the video
- ❌ Rushing through answers without scrolling slowly
- ❌ Not showing all 5 mandatory queries
- ❌ Not showing embeddings in Firebase Console
- ❌ Typing queries incorrectly
- ❌ Skipping the context chunks demonstration
- ❌ Audio too quiet or unclear
- ❌ Screen content too small to read
- ❌ Showing sensitive information (API keys, passwords)

### If Something Goes Wrong
- **Query doesn't return results**: Wait 2 minutes for embeddings to process, then retry
- **Firebase Console not loading**: Refresh the page and log in again
- **Recording software crashes**: Have backup recording software ready
- **Made a mistake**: Pause, take a breath, and restart that section (you can edit later)

## Troubleshooting

### Embeddings Not Showing
**Problem**: Firebase Console shows chunks but no embeddings  
**Solution**: Wait 2 minutes for the background scheduler to process embeddings, then refresh

### Query Returns No Results
**Problem**: Queries return empty responses  
**Solution**: Ensure embeddings are generated (check Firebase Console), wait and retry

### Upload Fails
**Problem**: PDF upload doesn't complete  
**Solution**: Check file size (< 10MB), verify internet connection, try again

### Authentication Issues
**Problem**: Can't sign in with Google  
**Solution**: Verify Firebase Auth is enabled, check browser console for errors

### Recording Quality Issues
**Problem**: Video is blurry or laggy  
**Solution**: Lower screen resolution, close other applications, use hardware encoding
